Servidor Docker para desenvolvedores

Se você chegou até aqui PARABÉNS, seu servidor está funcionando com o apache e o php e está pronto para que você consiga trabalhar no desenvolvimento de seus projetos.

A configuração que você encontrará aqui será a seguinte:

Para o apache você tem:

  • Um arquivo chamado ambientes.conf que é onde você poderá configurar os virtualhosts para o apache reconhecer, ele está mapeado para o conf.d e todas as alteraçãs feitas nele o docker reconhecerá.
  • A pasta logs é um mapeamento da pasta /var/log/apache2 do Docker, ou seja, nela você terá todos os arquivos de log do apache.

Para o php você tem:

  • Assim como no apache o php tem um arquivo na pasta conf chamado ambiente.ini, que tem o papel de ser lido como a configuração do php.ini do Docker.

O sistema todo tem uma pasta chamada APP, vista na imagem acima, e nela você pode colocar todo o seu código, que ele já reconhece o php para ser acessado no seu navegador.
